Maven工具入门

Posted on 2018-07-30
Words count in article: 1,671 字 | Reading time ≈ 7 分钟

简介

Maven是一个项目管理工具，是apache下的一个开源项目，纯Java开发，仅用于管理Java项目。量级同eclipse。

功能

编译、测试（JUnit）、运行、打包、部署。

优势

1.依赖管理：Maven项目占用的空间比非Maven项目小得多。因为Maven项目中没有jar包，jar包与项目分离。
2.一键构建：一个tomcat:run就能把项目运行起来。
3.可以跨平台。
4.应用于大型项目，可以提高开发效率。
5.可以分模块。

仓库

三种仓库：

本地仓库 自己维护
远程仓库 公司维护
中央仓库 maven团队维护

本地仓库没有的文件，可以从远程仓库（私服）下载，如果没有远程，会自动从中央仓库下载。如果有远程仓库但远程仓库没有所需的文件，远程仓库会自动从中央仓库下载文件，存到远程仓库再交给本地仓库。
中央仓库中jar包的数量大约是2亿（2015年），基本不会遇到用到的jar包中央仓库没有的情况。

maven项目目录结构

–src
|–main 主目录
| |–java java代码
| |–resources 配置文件（properties、xml）
| |–webapp
|–test 测试目录（JUnit测试）
| |–java java代码（JUnit测试）
| |–resources JUnit测试所用的配置文件（一般不放文件，所需的配置文件默认从main里面找）
–pom.xml maven项目的核心配置文件
如果不是这种目录结构，无法编译。

常用命令

compile 编译，只编译main目录下的文件
test 编译，只编译test目录下的文件，还会运行test的方法
package 打包，web项目打成war包，放在target目录下，名称取决于pom.xml文件
install 把项目发布到本地仓库（对web项目无意义），执行该命令会同时执行compile、test、package命令
deploy 发布到私服
clean 清理编译好的东西
set 生成项目说明的静态页面
tomcat:run 一键启动

依赖中，groupId、artifactId、version是坐标，scope是依赖范围，如果版本已经锁定，这里的version不起作用。
依赖范围
compile 编译、测试、运行都需要。打的war包里面会有该jar包，默认，可不写。
provided 编译（compile）时需要，测试时需要，运行时不需要。打的war包里面不存在该jar包（适用于jsp-api.jar、servlet-api.jar等）。
runtime 数据库驱动包，编译时不需要，测试时需要，运行时需要
test 针对JUnit，编译时不需要，测试时需要，运行时不需要
